Level Editor Design

About Levels

The Level Editor section is based on my project for the level design course, an advanced game design course I took at RIT.

Although it starts as an academic project, it is more than that. I put extra effort into those levels since I generally find passion in making them.

The levels shown below include a 2D platformer level - Battle Block Theater, a 3D puzzle level - Portal 2, and a 3D environmental storytelling level - Doom.

Battle Block Theater

More

This game editor has vibrant structure blocks (such as spikes, water, lasers, ships, fans, etc.) and diverse enemies (stealers, fire-breathing cats). Moreover, these modules can interact with each other. And that's is the starter of my level.

While facing a level, I prefer open worlds, so I want to give my player an impression of an opening feeling, having multiple ways to play through.

But the linear level usually is more controllable. Therefore, I used platforms with different heights and the game's jumping mechanism (you can jump higher if you run into water on fire) to make an opening scene place but actually a linear level.

Portal 2

More

Portal 2 is a famous puzzle game, however, since the player might have different experiences with the game, I designed two mini-teaching sections before the main stage.

In the first part, I use color cues- there are only two white blocks. Thereby guiding the player to discover that by using gravity, they can achieve a long-distance leap.

The second mini-part introduces the square switch and the optical bridge, respectively, which allows the player has a brief understanding of them. At the same time, I try to indicate a connection between the block switch and the light bridge.

Then, here comes the main stage; there are two puzzles in it. I set up a glass tunnel for the player, so they might walk through to check on the first puzzle from a different angle in case it's structure is too hard to understand. Then, the clue to solve the second puzzle is to use the light bridge to control the block switch. In order to recall that connection, I set the bridge and the block in the same sight again.

Doom

More

Doom's level editor has a built-in logic program, therefore, I use it to build an ambient narrative level rather than a normal shooting level. (Well, I admit it is also because I am so bad on FPS games).

During the overall gameplay, I use a lot of light guidance and camera guidance.

For example, in the beginning, I placed a bed on the player's left to block 60% of playerâ€™s vision. At the same time, nothing on the "right way", but a prominent red door, full of EXIT signs, with a short-circuit flashing light. Therefore, in the beginning, even if the player has not seen any text introduction, they can find the final goal - the exit.
